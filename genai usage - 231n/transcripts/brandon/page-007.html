## Route Both Arms Through `lib/train_4m.py`

### Context
`run_scaling_experiment_2.py` was calling `torchrun … run_training_4m.py` directly as a subprocess. Subprocess boundaries don't carry Python monkey-patches, so the neural integration in `lib/train_4m.py` (modality registration, trial sampling, presence masking) was completely bypassed. Both arms — control and treatment — should go through `lib/train_4m.py train` so the patch stack is always in effect.

### Changes Applied

**`run_scaling_experiment_2.py`** — two edits:
1. Add `import sys` to imports
2. Replace `launch_training` subprocess command:

```python
# Before
cmd = ["torchrun", "--nproc_per_node=1", "/opt/repo/ml-4m/run_training_4m.py",
       "--config", str(model_cfg_path)]
if test_run:
    cmd.extend(["--device", "cpu", "--dist_backend", "gloo"])

# After
cmd = [
    sys.executable,
    "/opt/repo/4m_training/lib/train_4m.py",
    "train",
    "--config", str(model_cfg_path),
]
if test_run:
    cmd.extend(["--", "--device", "cpu", "--dist_backend", "gloo"])
```

- `sys.executable` matches the same venv as the parent process
- `lib/train_4m.py train` applies the patches then calls `runpy.run_path("run_training_4m.py")` — the DDP env vars (`RANK`, `WORLD_SIZE`, etc.) are already set by `training_env()` so `os.environ.setdefault(...)` in `run_train` is a no-op
- `--` separator before extra args is stripped by `_clean_extra_argv` and the rest forwarded to `run_training_4m.py`

**`modal_image.py`** — one edit to `training_env()`:
```python

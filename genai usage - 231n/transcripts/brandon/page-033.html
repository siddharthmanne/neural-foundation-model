Yes — the depth GT shown in the figure is the pre-tokenized depth codes loaded from disk (tok_depth/shard_NNN.tar), then decoded back through the same DiVAE tokenizer.

The pipeline is:

Original depth map → DiVAE encoder → 196 int codes → stored in shard
                                                           ↓  (on disk)
                                                      tok_depth_t
                                                           ↓
                                          DiVAE decoder (25 DDIM steps)
                                                           ↓
                                                     "GT depth" image

So the GT isn't the original depth image — it's the tokenizer's reconstruction of it. This is intentional: it makes the GT and predicted depth directly comparable on the same tokenizer's output manifold (any quality floor from tokenization is equal for both). The model predicts token indices, so the only meaningful comparison is between two decoded reconstructions from the same codebook.

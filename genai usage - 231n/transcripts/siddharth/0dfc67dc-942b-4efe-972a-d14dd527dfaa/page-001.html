19	## 2. What 4M expects from a new modality — the binding constraint
20	
21	Before designing anything, the codebook size we ship has to fit 4M's modality
22	registration model. Read [`../../external/ml-4m/fourm/data/modality_info.py`](../../external/ml-4m/fourm/data/modality_info.py)
23	— that file is the source of truth.
24	
25	Concrete facts that drive the design:
26	
27	- **No fixed vocab across modalities.** Each modality declares its own
28	  `vocab_size` in `MODALITY_INFO`. Image-token modalities range from 4096
29	  (semseg) to 16384 (RGB); text/sequence is 30_000; `human_poses` "borrows"
30	  the 30k text vocab via `shared_vocab: ['caption']`. We are free to pick.
31	- **But vocab IS fixed at registration time.** 4M allocates a learnable
32	  embedding `[vocab_size, d_model]` for the modality. Adding tokens
33	  mid-training breaks that table. **Pick V before computing the token cache
34	  that 4M trains on; commit to it.**
35	- **`type: 'seq'` is the right modality type for MEG**, not image-grid.
36	  Mirror `human_poses` (`modality_info.py:172`) more than `tok_rgb`. The
37	  per-channel-per-timestep token stream is a sequence, not a 2D grid.
38	  Pose-tokenizer registration template:
39	  `vocab_size`, `encoder_embedding = SequenceEncoderEmbedding(...)`,
40	  `decoder_embedding = SequenceDecoderEmbedding(...)`,
41	  `max_length`, `num_channels`, `type: 'seq'`,
42	  `id: generate_uint15_hash('meg_tokens')`.
43	- **Multi-codebook (`num_codebooks > 1`) is supported by the `memcodes`
44	  quantizer** (used by `human_poses`, 1024 × 8). Cho 2026's tokenizers use
45	  a single codebook per channel. Stick with **`num_codebooks = 1`** for MEG
46	  — there is no reason to fight the upstream tokenizer's design.
47	- **Whether to share vocab with another modality is a 4M-side decision.**
48	  Human_poses reuses the text vocab because pose tokens are mapped into the
49	  WordPiece range; this lets a 4M decoder generate poses with the same head
50	  as captions. We do **not** want that for MEG — it couples MEG to text
51	  training. **Register a dedicated `meg_tokens` modality with its own
52	  vocab.**
53	
54	So the headline constraint: pick `V` and `max_length` (= per-trial token
55	sequence length) when you write the tokenizer, because the modality
56	registration that goes into 4M has to declare both.
57	
58	Sensible starting points, derived from Cho 2026 + the verified THINGS-MEG
59	data spec in §3 + 4M conventions:

141	## 4. Why three phases, in this order
142	
143	The phases trade off cost vs. expected fidelity. We deliberately start with
144	the cheapest possible MEG tokenizer to (a) prove the §5 harness is correctly
145	wired for MEG-shaped data, (b) put a floor on the leaderboard, and (c) avoid
146	prematurely committing to a deep architecture before we know what "good
147	enough" looks like on this specific THINGS-MEG distribution.
148	
149	| Phase | Tokenizer                                | Learns? | Cost     | Role                         |
150	|-------|------------------------------------------|---------|----------|------------------------------|
151	| 1     | μ-transform (Cho 2026 baseline)          | No¹     | seconds  | Floor + harness sanity check |
152	| 2     | Cho 2026 learnable AE (causal / non-causal) | Yes  | hours-GPU | Likely production tokenizer |
153	| 3     | BrainOmni tokenizer, finetuned           | Yes     | hours-GPU | Stretch / comparison         |

160	## 5. What "recalibration" / "finetuning" actually mean here
161	
162	The Cho 2026 paper warns that both tokenizer families need dataset-specific
163	adaptation. The mechanism is different for each.
164	
165	### Phase 1 — μ-transform recalibration (no gradient descent, but still a `fit` step)
166	
167	The μ-transform pipeline (paper §3.3.1):
168	
169	1. **Clip** the input to a percentile range `[q_lo, q_hi]` to suppress
170	   outliers.
171	2. **Max-abs scale** to `[−1, 1]`: `x' = x / s` with `s = max|x|`.
172	3. **μ-law compand**: `F(x') = sgn(x') · ln(1 + μ|x'|) / ln(1 + μ)`.
173	4. **Uniform bin** the compressed values into `V` bins → integer token IDs.
174	
175	The reference config Cho 2026 ships
176	([`../../external/Cho2026_Tokenizer/models/tokenizer/mu_transform/config.yml`](../../external/Cho2026_Tokenizer/models/tokenizer/mu_transform/config.yml))
177	is:
178	
179	```yaml
180	mu: 255
181	n_tokens: 256
182	normalization: max_abs
183	```
184	
185	There are **no learned weights**, but two parameters are dataset-dependent
186	and must be fit on a THINGS-MEG training split, frozen, and reused at
187	inference:
188	
189	- `s = max|x|`: the per-channel (recommended) or global max-abs scaler.
190	- `q_lo, q_hi`: clip thresholds (defaults around the 0.5 / 99.5 percentiles).
191	
192	`μ` and `V` are hyperparameters, not fit from data — start with μ=255,
193	V=256.
194	
195	Concretely: there is no `.train()` loop. There is a `fit(train_dataset) →
196	calibration.json` step that produces a small JSON sidecar with
197	`{s_per_channel, q_lo, q_hi, mu, V, channel_mode}`. At inference we load
198	that JSON and apply the stateless transform. Treat the calibration JSON as
199	a versioned tokenizer checkpoint.
200	
201	> Verified §3 fact that matters here: all 271 channels are `mag`, so the
202	> grad-vs-mag amplitude disparity problem does not apply. Per-channel `s`
203	> is still the safer default (inter-sensor amplitude variance is real even
204	> within one channel type), but the bin-saturation risk is much lower than
205	> it would be on a mixed-sensor dataset.
206	
207	### Phase 2 — Cho 2026 learnable tokenizer (actual gradient training)
208	
209	The Cho 2026 learnable tokenizer (paper §3.2) is an **autoencoder**, not a
210	VQ-VAE: encoder is GRU → Dense → LayerNorm producing per-time-step logits
211	over a vocab of size `V`; decoder is a bank of `V` 1-D conv kernels (causal
212	or non-causal) summed with learned per-token gains. Trained end-to-end with
213	MSE + an annealed soft-to-hard argmax relaxation, then unused tokens are
214	pruned ("token refactorization").
215	
216	Reference configs they ship
217	(`../../external/Cho2026_Tokenizer/models/tokenizer/{causal,noncausal}/config.yml`):
218	
219	```yaml
220	sequence_length: 200
221	n_channels: 52         # NB: 52 source-parcellated channels, NOT sensor
222	n_tokens: 128
223	token_dim: 10
224	rnn_n_units: 128
225	token_kernel_padding: causal  # or `same` for non-causal
226	training: { lr: 1e-5, batch: 32, epochs: 40, anneal_stages: 40 }
227	```
228	
229	Three things the configs tell us, in light of the verified data spec:
230	
231	1. **They trained on source-reconstructed data (52 parcels), not sensor
232	   space.** Our THINGS-MEG is **271-sensor**. The tokenizer is applied
233	   per-channel and the GRU has no channel-conditioning, so weights *can*
234	   be reused on sensor data, but the *amplitude distribution* of source
235	   parcels (after `osl_dynamics.standardize()`) is not identical to sensor
236	   amplitudes. This is the symmetric MEG version of the LABRAM/THINGS
237	   distribution mismatch in the parent doc.
238	2. **Their sequence_length = 200 samples ≈ 0.8 s.** Our trials are 281
239	   samples (1.4 s). Either reduce our windows to 200 samples (matches their
240	   training distribution → reuse pretrained weights cleanly) or retrain
241	   with `sequence_length = 281`. Strong prior: **window to 200** for 2a/2b
242	   to keep the pretrained encoder in-distribution; only change
243	   `sequence_length` for 2c.
244	3. **Their LR (1e-5) is very small.** Tells us finetuning is intended to be
245	   conservative — they expect the loss landscape to be flat around the
246	   pretrained optimum. Start with that LR and only increase if the loss is
247	   not moving after 1 epoch.
248	
249	"Finetuning on our dataset" then has three concrete recipes:
250	
251	- **2a.** Load pretrained weights, apply on THINGS-MEG **as-is** (zero
252	  finetune). Establishes the floor for what their pretrained tokenizer
253	  buys us on distribution-shifted data.
254	- **2b.** Load pretrained weights, continue MSE training on THINGS-MEG
255	  (lr=1e-5, anneal κ already at 0 → hard tokens — or restart anneal).
256	  This is the "actual finetune."
257	- **2c.** Take their architecture, train from scratch on THINGS-MEG. This
258	  is needed if (a) pretrained weights fail the §5 harness even after 2b,
259	  (b) we want to change `sequence_length` or `n_tokens`, or (c) the
260	  source-vs-sensor mismatch turns out to dominate. With ~108k trials
261	  pooled across P1–P4 (see §3 — much more than the parent doc estimated),
262	  from-scratch is more viable than first assumed.
263	
264	Run **2a → 2b → 2c** in order; stop as soon as the harness numbers are
265	acceptable. Do not skip 2a — the gap between 2a and 2b is the answer to
266	"how much did pretraining buy us."
267	
268	> System-level note: their pretraining data is **resting-state** MEG;
269	> THINGS-MEG is **event-related** (stimulus-locked). Expect the pretrained
270	> kernels to capture continuous oscillatory structure well and to
271	> underweight evoked transients. Reconstruction MSE may look fine while
272	> the §5.3 linear probe quietly tanks — watch that metric closely after
273	> each of 2a/2b/2c.
274

349	## 8. Evaluation contract
350	
351	Identical to the parent §5. The same `test_tokenizer.py` harness must pass
352	for each of the three MEG tokenizers before it is wired into 4M:
353	
354	1. Reconstruction fidelity (time MSE + per-channel Pearson + **spectral
355	   MSE**). Spectral MSE is the diagnostic metric for MEG — band power
356	   (especially α/β) is what carries task signal. PSD range: 1–100 Hz
357	   (Nyquist for 200 Hz data is 100 Hz).
358	2. Codebook utilization (perplexity, dead-code fraction). The μ-transform's
359	   dead-code fraction is the metric to watch; Cho 2026 prunes dead codes
360	   ("token refactorization") so its effective `V*` can be reported
361	   alongside nominal `V`.
362	3. Linear probe on THINGS labels. With 22,449 image-level codes / 1854
363	   concepts / ~27 superordinate categories, use the THINGS
364	   **superordinate** categories as the primary probe target, 1854-way
365	   concept ID as a secondary, image ID never (too noisy at 27k trials per
366	   subject). The `image_id → concept_id → superordinate` mapping comes
367	   from THINGS metadata, not the .fif files — add it under
368	   `meg/data.py`.
369	4. Token-sequence statistics (unigram entropy, bigram conditional entropy,
370	   run-length). The μ-transform is at particular risk of degenerate run
371	   lengths on slow drifts — watch this.
372	
373	MEG-specific harness parameters (PSD frequency range, named bands) go in
374	`meg_config.py` and are passed into the harness as `EvalConfig`. Do not
375	hardcode them.

def split_by_image(
    image_ids_per_trial: np.ndarray,
    defaults: SplitDefaults,
) -> Splits:
    """Partition trial indices into train/val/test grouped by image identity.

    Args:
        image_ids_per_trial: (n_trials,) int array of THINGS image codes,
            one entry per trial in the subject's epoch list. Equivalent to
            `epochs.events[:, 2]` from MNE.
        defaults: SplitDefaults with (train_frac, val_frac, test_frac, seed).

    Returns:
        Splits with .train, .val, .test arrays of trial indices. .val is a
        length-0 array if defaults.val_frac == 0.

        (test_frac, seed) — but DIFFERENT val_frac values — yields the same
        .test array. This is what guarantees Phase 1 and Phase 2 evaluate on
        the same held-out trials.
    """
    if image_ids_per_trial.ndim != 1:
        raise ValueError(
            f"image_ids_per_trial must be 1-D, got shape {image_ids_per_trial.shape}"
        )

    unique_images = np.unique(image_ids_per_trial)
    n_images = len(unique_images)

    # Deterministic shuffle of image IDs by seed only — independent of
    # val_frac. This is what enforces the cross-phase test-set invariant.
    rng = np.random.default_rng(defaults.seed)
    shuffled = unique_images.copy()
    rng.shuffle(shuffled)

    n_test = _round_count(n_images, defaults.test_frac)
    n_val = _round_count(n_images, defaults.val_frac)
    n_train = n_images - n_test - n_val
    if n_train <= 0:
        raise ValueError(
            f"After taking test ({n_test}) and val ({n_val}) out of {n_images} "
            f"unique images, no train images left."
        )

    # identity depends only on (seed, test_frac), not on val_frac.
    train_images = shuffled[:n_train]
    val_images = shuffled[n_train : n_train + n_val]
    test_images = shuffled[n_train + n_val :]

    train_idx = _trials_for_images(image_ids_per_trial, train_images)
    val_idx = _trials_for_images(image_ids_per_trial, val_images)
    test_idx = _trials_for_images(image_ids_per_trial, test_images)
    return Splits(
        train=train_idx,
        val=val_idx,
        test=test_idx,
        image_ids_test=np.sort(test_images),
    )

    return int(np.clip(round(frac * n_total), 0, n_total))

Where numpy vs torch goes (and why)

There are three layers in the pipeline. Only the middle one matters for GPU throughput:

Layer Lib Why
Disk → RAM (MNE reading .fif) numpy MNE has no torch backend; it always returns float64 numpy. There's no way around this.

        xs.append(X_s)

    """Apply the three encoder steps and return a tensor in [-1, 1].

    Args:
        x:     (B, C, T) float tensor on any device.
        calib: fitted MuCalibration (per-channel clip + scaler, μ).

    Returns:
        x_companded: (B, C, T) float in [-1, 1], same device/dtype as `x`.
    """

    # 1) Clip to per-channel [q_lo, q_hi]. Broadcast: (1, C, 1).
    lo = calib.clip_lo.to(x.device).view(1, -1, 1)
    hi = calib.clip_hi.to(x.device).view(1, -1, 1)
    x_clipped = torch.clamp(x, min=lo, max=hi)

"""Uniform binning of [-1, 1] floats into integer token IDs.

Companded values land in [-1, 1] (modulo clipping numerics, which we re-clamp
to be safe). We split that interval into V equal-width bins and emit the bin
index as the token ID.

Symmetric: bin centers are {-1 + (i + 0.5) * 2/V} for i in [0, V).

Inverse mapping (`dequantize`) returns the bin center, which is what the
decoder's inverse-μ-law expects.
"""

from __future__ import annotations

import torch


def quantize(x_companded: torch.Tensor, vocab_size: int) -> torch.Tensor:
    """Map (B, C, T) floats in [-1, 1] to (B, C, T) long token IDs in [0, V).

    Values outside [-1, 1] are clamped first — this handles tiny numerical
    drift from the encoder's float arithmetic without silently producing
    out-of-range tokens.
    """
    _assert_vocab_size(vocab_size)
    x = x_companded.clamp(-1.0, 1.0)
    # Map [-1, 1] -> [0, V) via affine: idx = floor((x + 1) / 2 * V), clamped.
    half_V = 0.5 * vocab_size
    idx = (x + 1.0) * half_V
    return idx.floor().clamp_(0, vocab_size - 1).to(torch.long)


def dequantize(tokens: torch.Tensor, vocab_size: int) -> torch.Tensor:
    """Map token IDs back to bin-center floats in [-1, 1].

    bin_center(i) = -1 + (i + 0.5) * 2 / V
    """
    _assert_vocab_size(vocab_size)
    bin_width = 2.0 / vocab_size
    return (tokens.to(torch.float32) + 0.5) * bin_width - 1.0


def bin_centers(vocab_size: int, dtype: torch.dtype = torch.float32, device=None) -> torch.Tensor:
    """The V bin centers as a 1-D tensor of shape (V,). Useful for plots / tests."""
    _assert_vocab_size(vocab_size)
    bin_width = 2.0 / vocab_size
    idx = torch.arange(vocab_size, dtype=dtype, device=device)
    return (idx + 0.5) * bin_width - 1.0

    if vocab_size < 2:

"""μ-transform decoder: bin centers → inverse μ-law → inverse max-abs scale.

We do NOT undo clipping — that's lossy by construction; any value clipped at
encode time is gone. The decoder's job is to map a token ID back to the
amplitude that best represents its bin in the original signal scale.

Inverse μ-law:
    F^-1(y) = sgn(y) * ((1 + μ)^|y| - 1) / μ
"""

from __future__ import annotations

import torch

from .calibration import MuCalibration
from .quantizer import dequantize


def decode(tokens: torch.Tensor, calib: MuCalibration) -> torch.Tensor:
    """Map (B, C, T) long tokens back to (B, C, T) float on the input scale.

    Args:
        tokens: long tensor in [0, V), shape (B, C, T).
        calib:  MuCalibration used at encode time.

    Returns:
        x_hat: (B, C, T) float on the same device/dtype family as calib tensors.
    """
    _assert_shape(tokens, calib)

    # 1) Token -> bin center in [-1, 1].
    y = dequantize(tokens, calib.vocab_size).to(calib.scaler.dtype)
    y = y.to(tokens.device if tokens.is_floating_point() is False else y.device)
    # Move calibration tensors to the token device for the rest of the math.
    device = tokens.device
    y = y.to(device)

    # 2) Inverse μ-law expansion back to [-1, 1] linear amplitude.
    mu = float(calib.mu)
    abs_y = y.abs()
    expanded = (torch.pow(1.0 + mu, abs_y) - 1.0) / mu
    x_norm = torch.sign(y) * expanded

    # 3) Undo max-abs scaling: x_hat = x_norm * s.
    s = calib.scaler.to(device).view(1, -1, 1)
    return x_norm * s

    Args:

"""Unit tests for splits.py.

The most important guarantee here is the cross-phase invariant: Phase 1 (μ)
and Phase 2 (Cho2026) must evaluate on identical test trials, even though
they call `split_by_image` with different val_frac. The first test below is

import numpy as np
import pytest

from meg.splits import Splits, split_by_image

@pytest.fixture
def image_ids_per_trial() -> np.ndarray:
    """Realistic shape: ~22k unique image IDs spread across ~27k trials.

    THINGS has ~22,449 unique images and ~27,048 trials per subject — most
    images appear once, some appear twice. We mimic that distribution at
    smaller scale so tests stay fast.
    """
    rng = np.random.default_rng(0)
    n_images = 1000
    n_trials = 1200
    image_pool = np.arange(n_images)
    chosen = rng.choice(image_pool, size=n_trials, replace=True)
    return chosen


@pytest.fixture
def mu_defaults() -> SplitDefaults:
    return SplitDefaults(train_frac=0.90, val_frac=0.0, test_frac=0.10, seed=0)


@pytest.fixture
def learnable_defaults() -> SplitDefaults:
    return SplitDefaults(train_frac=0.80, val_frac=0.10, test_frac=0.10, seed=0)


# ----------------------------- the critical invariant ---------------------


def test_test_set_identical_across_val_fractions(image_ids_per_trial, mu_defaults, learnable_defaults):
    """Same (seed, test_frac), different val_frac → SAME test image IDs.

    This is what makes the §5 leaderboard comparable across Phase 1 and
    Phase 2+. If this test fails, mu-transform and Cho2026 will be graded
    on different held-out images and we'll silently get an unfair comparison.
    """
    s_mu = split_by_image(image_ids_per_trial, mu_defaults)
    s_learn = split_by_image(image_ids_per_trial, learnable_defaults)
    np.testing.assert_array_equal(s_mu.image_ids_test, s_learn.image_ids_test)


def test_test_set_changes_with_seed(image_ids_per_trial, mu_defaults):
    """Same params except seed → different test set. Sanity check on the rng path."""
    seed_a = split_by_image(image_ids_per_trial, mu_defaults)
    seed_b = split_by_image(
        image_ids_per_trial,
        SplitDefaults(train_frac=0.90, val_frac=0.0, test_frac=0.10, seed=42),
    )
    assert not np.array_equal(seed_a.image_ids_test, seed_b.image_ids_test)


# ----------------------------- by-image invariants ------------------------


def test_image_partitions_are_disjoint(image_ids_per_trial, learnable_defaults):
    """No image appears in more than one of train/val/test."""
    s = split_by_image(image_ids_per_trial, learnable_defaults)
    train_imgs = set(image_ids_per_trial[s.train].tolist())
    val_imgs = set(image_ids_per_trial[s.val].tolist())
    test_imgs = set(image_ids_per_trial[s.test].tolist())
    assert train_imgs.isdisjoint(val_imgs)
    assert train_imgs.isdisjoint(test_imgs)
    assert val_imgs.isdisjoint(test_imgs)


def test_no_trial_appears_twice(image_ids_per_trial, learnable_defaults):
    """Trial indices are unique across the three sets — no leakage."""
    s = split_by_image(image_ids_per_trial, learnable_defaults)
    combined = np.concatenate([s.train, s.val, s.test])
    assert len(combined) == len(set(combined.tolist()))


def test_all_trials_covered(image_ids_per_trial, learnable_defaults):
    """Every trial ends up in exactly one of train / val / test."""
    s = split_by_image(image_ids_per_trial, learnable_defaults)
    combined = np.sort(np.concatenate([s.train, s.val, s.test]))

def test_split_sizes_roughly_match_requested_fractions(image_ids_per_trial, learnable_defaults):

    # Allow caller-provided seed to override the dataclass default.
    split_cfg = SplitDefaults(
        train_frac=MU_SPLIT_DEFAULTS.train_frac,
        val_frac=MU_SPLIT_DEFAULTS.val_frac,
        test_frac=MU_SPLIT_DEFAULTS.test_frac,
        seed=seed,

def _build_mu_transform(calibration_payload: dict[str, Any]):

@app.function(

    cpu=4.0,
    memory=32 * 1024,
    timeout=60 * 60,

def evaluate_remote(
    tokenizer_name: str,
    tokenizer_payload: dict[str, Any],
    n_test: int = 3000,
    seed: int = 0,
    use_mu_split: bool = True,
) -> dict[str, Any]:
    """Run the §5 four-axis harness against the named tokenizer.

    Args:
        tokenizer_name: which factory to call (e.g. "mu_transform").
        tokenizer_payload: tokenizer-specific config (e.g. calibration dict

        seed: split seed. Must match the seed used at calibration time.
        use_mu_split: True for Phase-1 (no val), False for learnable phases.

        Small enough to print or post-process locally.

    import numpy as np
    import torch

    from evaluation import EvalConfig, evaluate
    from neural_tokenizers.meg import (
        EVAL_DEFAULTS,
        LEARNABLE_SPLIT_DEFAULTS,
        MU_SPLIT_DEFAULTS,
        SplitDefaults,
    )
    from neural_tokenizers.meg.data import list_subjects, load_trials_pooled
    from neural_tokenizers.meg.splits import split_by_image

    if tokenizer_name not in _TOKENIZER_FACTORIES:
        raise ValueError(
            f"unknown tokenizer {tokenizer_name!r}; "
            f"available: {sorted(_TOKENIZER_FACTORIES)}"
        )

    base_split = MU_SPLIT_DEFAULTS if use_mu_split else LEARNABLE_SPLIT_DEFAULTS
    split_cfg = SplitDefaults(
        train_frac=base_split.train_frac,
        val_frac=base_split.val_frac,
        test_frac=base_split.test_frac,
        seed=seed,
    )

    subjects = list_subjects()
    print(f"[eval] discovered subjects: {[s.subject for s in subjects]}")

    # Per-subject test indices.
    test_per_subj: dict[str, np.ndarray] = {}
    total_test = 0
    for s in subjects:
        sp = split_by_image(s.image_ids, split_cfg)
        test_per_subj[s.subject] = sp.test
        total_test += len(sp.test)
    print(f"[eval] total test trials available: {total_test}")

    # Cap at n_test if requested.
    if n_test > 0 and total_test > n_test:

        rng = np.random.default_rng(seed)
        for k, idx in test_per_subj.items():
            if len(idx) > per_subj_cap:
                pick = rng.choice(len(idx), size=per_subj_cap, replace=False)
                test_per_subj[k] = np.sort(idx[pick])
        total_test = sum(len(v) for v in test_per_subj.values())
        print(f"[eval] capped to {total_test} test trials")

    X, image_ids, _ = load_trials_pooled(subjects, test_per_subj)
    print(f"[eval] materialized X={tuple(X.shape)} dtype={X.dtype}")

    # Build the tokenizer locally inside the container.
    factory = _TOKENIZER_FACTORIES[tokenizer_name]
    tok = factory(tokenizer_payload)

    # numbers should be interpreted relative to the in-trial random baseline
    # the harness also computes, NOT as absolute concept accuracy yet.
    _, dense_labels = np.unique(image_ids, return_inverse=True)
    labels = torch.from_numpy(dense_labels.astype("int64"))

    device = "cuda" if torch.cuda.is_available() else "cpu"
    X = X.to(device)
    print(f"[eval] running harness on device={device} ...")
    cfg = EvalConfig(
        sample_rate_hz=EVAL_DEFAULTS.sample_rate_hz,
        bands=dict(EVAL_DEFAULTS.bands),
        device=device,
        batch_size=64,
        seed=seed,
        psd_nperseg=EVAL_DEFAULTS.psd_nperseg,
        probe_epochs=EVAL_DEFAULTS.probe_epochs,
        probe_top_k=EVAL_DEFAULTS.probe_top_k,
        probe_test_frac=EVAL_DEFAULTS.probe_test_frac,
    )
    report = evaluate(tok, X, labels, cfg)
    print(report)

    return {
        m.name: dict(m.values)
        for m in (report.reconstruction, report.codebook, report.sequence, report.probe)
        if m is not None
    }

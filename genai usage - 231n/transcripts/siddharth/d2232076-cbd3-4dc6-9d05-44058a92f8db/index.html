#12026-06-03T01:48:35.925Z

rn what is being inputted into 4m in terms of neural modalities? is it raw integer indices, or is it the codebook embeddings?

3 bash · 2 read

Clear answer. Here's what's actually fed into 4M for the neural modalities:

Raw integer token IDs — not codebook embeddings.

The on-disk shards store integer codes from the RVQ/VQ tokenizers, the dataloader passes them through NeuralTargetTransform (just clips to code_max and casts to int64), and they enter 4M as d["tensor"] of shape (B, P) integer indices.

Inside 4M, each modality's own nn.Embedding(vocab_size, dim_tokens) table — owned by the 4M model, not the tokenizer's codebook — turns those ids into vectors. See fourm_neural_embeddings.py:109 (encoder) and fourm_neural_embeddings.py:148 (decoder), called inside forward / forward_embed at fourm_neural_embeddings.py:120 and fourm_neural_embeddings.py:159.

Concretely per modality: - MEG: 4 modalities tok_meg_rvq0..3, each (B, 128) int ids, vocab 512, mapped via a fresh nn.Embedding(512, dim_tokens) + axial position (learned source + sincos time). - EEG: tok_eeg, (B, 17) int ids, vocab 8192, mapped via nn.Embedding(8192, dim_tokens) + 1D sincos pos.

Why this design (system-level): 4M's masked prediction needs to read AND write the same modality — meaning the same vocab_size-way logit head has to invert the input mapping. Feeding the raw tokenizer codebook embeddings (frozen, ≠ dim_tokens) would force a projection and decouple input/output spaces, breaking the symmetric encoder↔decoder weight tying pattern used in 4M for image tokens. By keeping the tokenizer purely as a discretizer and letting 4M relearn an embedding table at its own width, MEG/EEG slot in exactly like tok_rgb — which is also why pretokenized: True is set in the modality info.

#22026-06-03T01:50:09.427Z

so its training a codebook from scrtch for each modality?

Yes — exactly.

For each neural modality, 4M instantiates a fresh nn.Embedding(vocab_size, dim_tokens) on the encoder side and a separate fresh nn.Embedding(vocab_size, dim_tokens) + nn.Linear(dim_tokens, vocab_size) logit head on the decoder side. All randomly initialized, all trained end-to-end with the rest of 4M. The BrainOmni/LaBraM codebook vectors are never loaded — only the discrete ids they produced.

So per run, 4M learns from scratch: - MEG: 4 × (512-entry encoder table + 512-entry decoder table + 512-way logit head), one per RVQ layer - EEG: 1 × (8192-entry encoder table + 8192-entry decoder table + 8192-way logit head)

Two consequences worth noting:

  1. The tokenizer's geometry is discarded. RVQ codebook 0 and codebook 1 have a meaningful residual relationship in BrainOmni's latent space; 4M sees them as four independent vocabularies and has to rediscover any cross-layer structure through co-occurrence. That's the price of the symmetric-modality design.

  2. Encoder and decoder embeddings are not tied in this code (each side has its own token_emb). Stock 4M image modalities don't tie them either, so this matches convention — but it does mean 2× the embedding params per modality. For EEG that's ~2 × 8192 × 768 ≈ 12.6M params just for the lookup tables, which is non-trivial at small model sizes.

If you ever wanted to warm-start from the tokenizer's codebook, the seam would be in _NeuralGridEncoderEmbedding.init / _NeuralGridDecoderEmbedding.init — load the codebook tensor, project to dim_tokens, and copy into token_emb.weight. But you'd lose the clean symmetry with tok_rgb and need to decide whether to freeze or fine-tune.

Neither, strictly — and importantly not from decoded/reconstructed signal. The probe never calls decode_tokens. Here's what each feature set actually is:

For the "tokens_" features (the thing being evaluated): - If the tokenizer exposes tokens_to_embedding → embed the token IDs through the codebook lookup (BrainOmni: sum of RVQ codebook vectors per cell) and either mean-pool (probe.py:181) or keep the natural spatial layout for the CNN head (probe.py:222). - If it doesn't → bag-of-codes* histogram over the integer IDs (probe.py:229-235).

So the "tokens" path uses codebook embeddings or raw IDs — never x_hat.

The two bracketing anchors: - raw = the actual raw MEG signal x, flattened (linear/mlp) or kept as (C, T) (CNN). Not reconstructed — the literal input. - random = uniformly random token IDs, bag-of-codes'd.

Why this design (the §5.3 logic in your CLAUDE.md): the question the probe answers is "can 4M's transformer linearly read the stimulus from what it actually sees as input?" 4M sees codebook embeddings (well, its own re-learned embedding of the IDs — see your earlier question), not x_hat. Probing on decode_tokens(tokens) would measure something else entirely — "did the decoder preserve task signal in time-domain reconstruction" — which is a §5.1 reconstruction question, not a representation-quality question. Probing on the reconstruction would also smuggle the decoder's capacity into the score: a stronger decoder could rescue lossy tokens.

The raw anchor is the closest the harness comes to "what if we just used the signal directly" — it's the ceiling the tokens are trying to approach.

Q2: Probe on reconstructed raw

Short answer: probably yes if every modality lands in every batch on both the input and target side every step, but you should verify in the training code first — find_unused_parameters=True in 4M usually fires for two distinct reasons, only one of which is about "multi-trial" modalities. Short answer: not just based on "no multi-trial modalities" — the actual condition is stricter. Here's what's actually controlling that flag in our setup.

find_unused_parameters=True is required whenever any parameter can miss a gradient on a forward pass. In our 4M training, two independent causes can fire each step:

1. Presence flags zero out absent modalities. PresenceAwareUnifiedMasking.zero_absent_budgets at neural_masking.py:100 sets a modality's input AND target budgets to 0 when its presence flag is False. If even one sample in a global batch has e.g. MEG absent and no other sample on any rank routes that step's gradient through MEG's encoder+decoder embeddings, those params skip → unused.

2. The Dirichlet token-budget sampler can assign 0 tokens to a side of a modality even when the modality is present. With min_tokens=0 in our MODALITY_INFO (set for all neural modalities in fourm_neural_modalities.py:86,107) and the upstream Dirichlet allocator, getting input_budget=0 or target_budget=0 for a given modality is a regular occurrence, not a rare edge case. That zero kills the gradient through that side's token_emb + (for decoder) to_logits.

So:

  • "No modality has multiple trials" is not the relevant condition — it's about per-step token allocation, not per-sample trial count.
  • You can disable find_unused_params only if BOTH:
  • Every sample in every batch has every modality present (no zero-presence), AND
  • The sampler is configured so no modality ever gets 0 tokens on either side (e.g. min_tokens >= 1 enforced as a hard floor, or keep schemes that pin >=1 cell per side).

If either fails on any step, DDP will hang or error with "expected to mark a variable ready only once" or "unused parameter detected." With our current MEG/EEG presence-gated setup, both fail routinely.

Practical implication for our config: the find_unused_params: true in 4m_things_main.yaml:28 is load-bearing as long as we have: - Variable MEG/EEG presence per sample, OR - The default Dirichlet sampler with no nonzero floor.

If you want to flip it off, the cheapest path is to assert every sample has every modality (drop the presence mechanism, only train on the intersection of RGB ∩ MEG ∩ EEG image_ids) AND add a min_tokens=1 floor enforced in PresenceAwareUnifiedMasking so the Dirichlet allocation can't zero a side. Even then I'd recommend keeping it on initially and toggling off only after a full epoch runs cleanly with both conditions — DDP failures from this are silent until the wrong rank hits the wrong sample.

The speedup from find_unused_params=False is typically 5-15% wall time, so it's only worth chasing once everything else is stable.

\subsection{Neural Data Tokenizer Evaluation}
\label{subsec:tokenizer-eval}

Each tokenizer is evaluated on four independent axes that target
distinct failure modes (reconstruction fidelity, codebook health,
discriminability, and sequence learnability). To make each axis
interpretable we compare four feature sets per trial:
\textbf{raw} (the preprocessed MEG signal itself, an upper bound on
what any tokenizer could preserve), \textbf{random} (uniformly random
token IDs of the matching shape, a lower bound representing zero
information content), \textbf{tokens\_all} (the trained tokenizer's
output, with multi-codebook tokenizers' codebook embeddings summed
across all RVQ layers), and---for MEG, where BrainOmni uses
residual vector quantization with $Q{=}4$ layers---\textbf{tokens\_rvq0},
the coarsest RVQ layer alone, which asks whether top-level structure
carries the relevant signal independently of finer residual layers.
The raw/random brackets are essential: without them an absolute
probe number cannot be interpreted as ``the tokenizer preserved
signal'' versus ``the task is intrinsically hard from this modality.''

\paragraph{Reconstruction fidelity.} We measure time-domain mean
squared error, mean per-channel Pearson correlation, and the mean
squared error between Welch power spectral density estimates of $x$
and $\hat{x}$ in the 1--100\,Hz band. The spectral term is the most
diagnostic of the three: neural task signal lives in named frequency
bands ($\theta/\alpha/\beta/\gamma$), and a tokenizer can drive
time-domain MSE down while still scrambling band power.

\paragraph{Codebook health.} A tokenizer that achieves low
reconstruction error by collapsing onto a small subset of its codes
is broken even when MSE looks clean. We report perplexity
$\exp(H(p))$ of the empirical code distribution $p$, the
dead-code fraction (codes never selected on the test set), and the
unigram entropy $H(p)$ in nats. We additionally compute bigram
conditional entropy $H(t_i \mid t_{i-1})$ and mean run-length to detect
sequence-level collapse: a tokenizer that emits highly redundant
streams (e.g., long runs of a single code per trial) can pass the
unigram diagnostics while leaving the masked-prediction objective
nothing to learn.

\paragraph{Downstream probes.} The central test of whether the tokens
carry task-relevant signal is a probe trained on top of the frozen
tokenizer. We evaluate three tasks of decreasing chance level and
increasing difficulty:
\begin{itemize}
    \item \textbf{Subject identity} (4-way, chance 25\%): a pipeline
    sanity check. Subject identity is dominated by sensor
    geometry and head position and is trivially decodable from raw
    MEG; tokens that fail this task indicate a broken eval pipeline
    rather than a broken tokenizer.
    \item \textbf{Animacy} (2-way, chance 50\%): a sensitivity check.
    Living vs.\ non-living discrimination is well-established in
    MEG decoding and provides a coarse target that should be
    reachable from a competent tokenizer.
    \item \textbf{Category27} (27-way, chance 3.70\%): the
    publication-target gate. We coarsen the 1{,}854 THINGS concept
    labels into 27 superordinate categories from the THINGS canonical
    metadata. Only the 892 concepts with a clean single-category
    Bottom-Up label are retained, reducing the eval to roughly
    5{,}000 of the original $\sim$10{,}000 test trials (around
    185 trials per class); the remaining trials are dropped from
    this task only.
\end{itemize}

For each task we fit three probe heads: a single linear layer
(class-weighted cross-entropy logistic regression), a two-layer MLP
with ReLU and dropout (tests ``information present but nonlinearly
encoded''), and a small CNN that respects each feature set's natural
spatial--temporal layout (Conv2D over the $16\times 8$ latent--time
grid for BrainOmni tokens, Conv1D over (channel, time) for raw MEG).
All probes use 5-fold cross-validation with class-weighted cross-entropy
loss; we report balanced accuracy as the headline metric since its
chance floor is $1/n_{\text{classes}}$ regardless of class frequency,
unlike top-1 accuracy whose floor under imbalance is the majority
class frequency.

\paragraph{Model-free retrieval.} As a zero-trainable-parameter
complement to the probe, we run nearest-neighbour retrieval at $k{=}1$
and $k{=}5$ on cosine similarity over the tokenizer's continuous code
embeddings and on Jaccard overlap over the discrete bag of codes per
trial. Retrieval separates two otherwise-confounded failure modes: a
task on which both the probe and retrieval are at chance indicates
the signal is not in the token geometry at all, while a task on which
retrieval is at chance but the probe lifts above chance indicates the
signal is present but only linearly accessible after a learned
projection. Retrieval is reported on the same three tasks.

A tokenizer passes the gate for downstream foundation-model training
when (i) tokens reach the raw upper bound (within fold variance) on at
least one above-chance task, (ii) the codebook is not collapsed, and
(iii) tokens lift cleanly above the random lower bound on the gate
task (Category27). Tokenizers that pass reconstruction and codebook
diagnostics but stay at the random floor on the probe are the
characteristic failure mode this evaluation is designed to catch.

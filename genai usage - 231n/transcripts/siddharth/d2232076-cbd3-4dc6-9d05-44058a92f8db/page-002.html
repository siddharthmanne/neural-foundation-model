114	    Produces one signal per unique image — the canonical "cross-subject +
115	    cross-rep" averaging used by Experiment 2 (see
116	    ``notes/meg_tokenization.md``). The averaging axis collapses both
117	    subject and within-subject repetitions, so the output has no subject
118	    label per row.
119	
120	    Args:
121	        X: ``(N, C, T)`` torch or numpy array of single-trial signals.
122	        image_ids: ``(N,)`` int array, trial-aligned image codes.
123

125	        X_avg:     ``(M, C, T)`` torch.float32 tensor (M = unique image
126	                   count).

128	
129	    Memory-frugal: accumulates in float64 element-wise inside `np.add.at`
130	    but **never materializes a float64 copy of the input**. The 88k-trial
131	    training tensor (~27 GB float32) is the largest object in this
132	    pipeline; doubling it with an f64 cast would push the container OOM.
133	    """
134	    if isinstance(X, torch.Tensor):
135	        X_np = X.detach().cpu().numpy()  # no dtype cast — preserve source f32
136	    else:
137	        X_np = np.asarray(X)
138	
139	    image_ids = np.asarray(image_ids, dtype=np.int64)
140	    if X_np.shape[0] != image_ids.shape[0]:
141	        raise ValueError(
142	            f"X.shape[0]={X_np.shape[0]} must match image_ids length "
143	            f"{image_ids.shape[0]}"
144	        )
145	
146	    unique_ids, inverse = np.unique(image_ids, return_inverse=True)
147	    n_groups = len(unique_ids)
148	    C, T = X_np.shape[1], X_np.shape[2]
149	
150	    # Sums stay in f64 for accumulator precision when a single group has
151	    # 40+ contributors (test images × 4 subjects × 12 reps); numpy promotes

153	    sums = np.zeros((n_groups, C, T), dtype=np.float64)
154	    counts = np.zeros(n_groups, dtype=np.int64)
155	    np.add.at(sums, inverse, X_np)
156	    np.add.at(counts, inverse, 1)
157	
158	    X_avg_np = sums / counts.reshape(n_groups, 1, 1)

240	# ---------- subsampling for calibration -----------------------------------
241	
242	def sample_train_trials(
243	    subject_indices: list[SubjectIndex],
244	    train_indices_per_subject: dict[str, np.ndarray],
245	    n_sample: int,
246	    seed: int = 0,
247	    dtype: torch.dtype = torch.float32,
248	) -> torch.Tensor:
249	    """Uniformly subsample N trials from the pooled train split.
250	
251	    Used by μ-transform calibration: percentile + max-abs estimates converge
252	    fast in sample size, so loading a few hundred MB of trials is enough to
253	    fit the calibration parameters to high accuracy. Subsampling sidesteps
254	    the streaming-quantile complexity entirely (CLAUDE.md §5 / discussion).
255	
256	    Args:
257	        subject_indices: from list_subjects().
258	        train_indices_per_subject: per-subject train trial indices.
259	        n_sample: total trials to draw across all subjects, pooled.
260	        seed: rng seed.
261	        dtype: output torch dtype.
262	
263	    Returns:
264	        X: (n_sample, C, T) torch tensor on CPU. Caller moves to GPU.
265	    """
266	    # Build a flat list of (subject_pos, trial_idx) pairs, then sample.
267	    pairs: list[tuple[int, int]] = []
268	    for pos, s in enumerate(subject_indices):
269	        idx = train_indices_per_subject.get(s.subject)
270	        if idx is None:
271	            continue
272	        for t in idx:
273	            pairs.append((pos, int(t)))
274	    if not pairs:
275	        raise ValueError("No trials available to subsample (train_indices empty).")
276	
277	    rng = np.random.default_rng(seed)
278	    n_sample = min(n_sample, len(pairs))
279	    chosen = rng.choice(len(pairs), size=n_sample, replace=False)
280	
281	    # Group chosen pairs by subject for a single .fif read per subject.
282	    per_subj: dict[str, list[int]] = {}
283	    for k in chosen:
284	        pos, t = pairs[k]
285	        per_subj.setdefault(subject_indices[pos].subject, []).append(t)
286	
287	    xs: list[torch.Tensor] = []
288	    for s in subject_indices:
289	        ts = per_subj.get(s.subject)
290	        if not ts:
291	            continue
292	        X_s, _ = load_trials(s, np.asarray(ts, dtype=np.int64), dtype=dtype)
293	        xs.append(X_s)
294	    return torch.cat(xs, dim=0)

Since tokens stay (B, 16, 8, 4) and codebook_size=512 regardless of input channel count, random is unchanged. The thing that changes shape is only raw (now (B, C_occ, T)).

Two system-level notes worth flagging since they were design choices:

  1. Channel restriction lived at the data layer, not the model. BrainOmni's cross-attention from sensors → 16 fixed latents was already channel-count agnostic.

System-level point worth absorbing (the kind of thing your CLAUDE.md asks me to flag):

This was a silent-success bug, the most dangerous class. The shell exit code was 0, the file existed, the function returned cleanly — every signal said "done." The only way to catch it was to assert a post-condition about the result, not just trust the operation's return code.

General rule: for any download/extract/transfer step, end the function with an assertion of what should be true (file size > X, row count == Y, checksum matches). It's three extra lines and turns silent corruption into a loud failure. I added a 10 GB floor here — small enough to not have false alarms, large enough to catch this exact bug.

This is also a good candidate for a reusable verification helper if you end up writing more downloaders — assert_min_size(path, min_bytes) etc. — but not worth abstracting yet (one downloader).

  mne.read_epochs() and what we'll feed into the PyTorch DataLoader.
  We skip the .mat (60 GB, MATLAB duplicate of the same MEG data) and
  the eye-tracking .csv (33 GB, not needed for a first-pass MEG FM).

  Net: 33 GB instead of 127 GB, no WAF, anonymous, resumable.

Bytes transfer S3 -> Modal directly. Your laptop's only job is `modal run`.

Run:
    modal run --detach modal_download_things_meg.py::download
"""

import os
import subprocess

from modal_app import app, data_volume, image

# awscli for `aws s3 sync` with anonymous public-bucket access.
# add_local_python_source: ships modal_app.py into the container so the
# `from modal_app import ...` line above resolves at remote-runtime.
download_image = (
    image
    .pip_install("awscli")
    .add_local_python_source("modal_app")
)

S3_SOURCE = "s3://openneuro.org/ds004212/derivatives/preprocessed/"
DEST_DIR = "/data/things-meg/preprocessed"

# Post-condition floor. With 16 .fif files totaling ~33 GB on S3, anything
# under 25 GB after a successful sync means files are missing.
MIN_EXPECTED_BYTES = 25 * 1024 * 1024 * 1024


@app.function(
    image=download_image,
    volumes={"/data": data_volume},
    # Network-bound. CPU-only; a GPU would idle while burning credits.
    cpu=4.0,
    memory=8 * 1024,
    # 3 hours is generous for 33 GB; sync is resumable so any timeout
    # is recoverable by re-running.
    timeout=60 * 60 * 3,
)
def download():
    """aws s3 sync, .fif only, into /data/things-meg/preprocessed/."""
    os.makedirs(DEST_DIR, exist_ok=True)

    # --exclude "*" then --include "*.fif" is the standard aws-cli idiom
    # for "only grab files matching this pattern." Order matters: the
    # filters are applied left-to-right.
    subprocess.run(
        [
            "aws", "s3", "sync",
            "--no-sign-request",
            "--no-progress",
            "--exclude", "*",
            "--include", "*.fif",
            S3_SOURCE,
            DEST_DIR,
        ],
        check=True,
    )

    # Post-condition: verify total size is in the expected range. Without
    # this we have no defense against a silent partial sync (zero exit
    # but missing files).
    total = sum(
        os.path.getsize(os.path.join(DEST_DIR, f))
        for f in os.listdir(DEST_DIR)
        if os.path.isfile(os.path.join(DEST_DIR, f))
    )
    print(f"Synced {len(os.listdir(DEST_DIR))} files, total {total / 1e9:.2f} GB")
    if total < MIN_EXPECTED_BYTES:
        raise RuntimeError(
            f"Synced data is suspiciously small ({total / 1e9:.2f} GB). "
            f"Expected >{MIN_EXPECTED_BYTES / 1e9:.0f} GB. "
            f"Check that the source S3 prefix and include filter are correct."
        )

    subprocess.run(["du", "-sh", DEST_DIR], check=True)
    subprocess.run(["ls", "-la", DEST_DIR], check=True)
    data_volume.commit()

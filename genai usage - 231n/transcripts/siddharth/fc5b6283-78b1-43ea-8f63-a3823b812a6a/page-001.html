Good question to pause on before moving 377 GB around. Two things matter more than the 90/10 ratio:

1. Split now, but the axis matters more than the timing. With ~90k trials where the same 22,448 images are viewed many times (4 subjects × 12 sessions ≈ 48 repeats per image), a random 90/10 split will leak — the same image ends up in both train and val, and your model learns image-specific neural signatures rather than a generalizable representation. Val loss will look great and lie to you.

What you actually want depends on what generalization you're testing: - Held-out images (split the 22,448 image IDs 90/10, propagate to all trials) → tests "can the model encode novel stimuli." Most defensible default for a vision-conditioned MEG FM. - Held-out subjects (3 train / 1 val) → tests cross-subject transfer. Crucial if you eventually want zero-shot on new people, but harsh and noisy with only 4 subjects. - Held-out sessions (e.g., session 12 of each subject) → tests session/drift robustness. - Combination (e.g., held-out images × held-out sessions) → cleanest but smallest val set.

2. Don't physically split the files on Modal. Store all 377 GB as one flat dataset and define splits via a manifest (splits.json / splits.parquet: trial_id → "train" | "val", with the split seed/policy versioned). Reasons: - Re-splitting later means editing a manifest, not re-uploading hundreds of GB. - You can ship multiple split policies (image-holdout, subject-holdout) side-by-side and compare. - Reproducibility: the split is a versioned artifact, not implicit in directory layout.

This is also the system-design point worth internalizing: separate the physical storage layout from the logical dataset partition. Train/val isn't a property of the bytes, it's a property of the experiment.

On the 90/10 ratio: fine if val doubles as your downstream eval, but for pure pretraining-loss monitoring, 1–5% is plenty. With 22,448 images, a 10% image-holdout gives ~2,245 held-out images × ~48 repeats = ~108k held-out trials, which is more than you need just to watch a loss curve.

1	# When and how to write a Modal file
2	
3	For any teammate who needs to run something on Modal. If you don't need Modal for your current work (e.g., you're happy on FarmShare), skip this.
4	
5	## What Modal is, briefly
6	
7	Modal is a service that runs your Python code on remote computers — including ones with GPUs — and bills you per second of compute used. Instead of buying a GPU machine or fighting a queue, you write a Python script with a few decorator lines saying "run this on Modal," type `modal run my_file.py`, and Modal:
8	
9	1. Builds a Linux container with your Python and dependencies (the **Image**).
10	2. Attaches the GPU you asked for.
11	3. Mounts any shared storage you specified (the **Volume**).
12	4. Injects any API tokens you need (the **Secrets**).
13	5. Runs your code there.
14	6. Tears the container down.
15	
16	You see the logs streaming in your terminal as if it were running locally. The container is gone afterwards; anything you wrote to the Volume persists.
17	
18	Our team has $200 of shared credits in a Modal workspace called `neural-fm`. Any job you run inside that workspace draws from the shared pot.
19	
20	## What a Modal file does
21	
22	A Modal file is just a Python script with one extra ingredient: `@app.function(...)` decorators on the functions you want to run remotely instead of locally. The decorator tells Modal "this function isn't meant to run on the laptop — when called via `.remote()`, ship it to a container and execute it there."
23	
24	The four pieces every Modal file needs:
25	
26	| Piece | What it is | Where ours comes from |
27	|---|---|---|
28	| **App** | A label that groups related jobs in the dashboard | `app = modal.App("neural-fm")` in `modal_app.py` |
29	| **Image** | OS + Python packages your code needs | Defined in `modal_app.py` from `requirements.txt` |
30	| **Volume** (optional) | Persistent disk for things you want to keep between runs | `data_volume`, mounted at `/data` |
31	| **Secret** (optional) | API tokens (HF, W&B, etc.) without committing them to git | **You create your own** — see the section below |
32	
33	The App, Image, and Volume are shared infrastructure — `from modal_app import ...` and you have them. **Secrets are personal** and you create your own (see "Personal secrets" below). The reason: Modal secrets are workspace-scoped, meaning a secret defined in the shared `modal_app.py` would be usable by every teammate, which is not what you want for personal credentials.
34	
35	## Personal secrets (HF tokens, W&B keys, etc.)
36	
37	If your Modal job needs to authenticate with Hugging Face, Weights & Biases, or any other external service, **create your own Modal secret first**. Probable best not to share tokens via a generic secret name in the shared `modal_app.py`. Will update if this changes.
38	
39	One-time setup, from your laptop terminal:
40	
41	```
42	modal secret create hf-yourname HF_TOKEN=hf_xxxxx_your_token_here
43	```
44	
45	(`hf-liane`, `hf-brandon`, `hf-sid` — pick something obvious so we can tell whose token is whose in the dashboard.)
46	
47	Then in your Modal file:
48	
49	```python
50	import modal
51	from modal_app import app, image, data_volume
52	
53	my_hf = modal.Secret.from_name("hf-yourname")
54	
55	@app.function(image=image, volumes={"/data": data_volume}, secrets=[my_hf])
56	def my_job():
57	    import os
58	    token = os.environ["HF_TOKEN"]   # set automatically by the Secret
59	    ...
60	```
61	
62	Same idea for W&B (`modal secret create wandb-yourname WANDB_API_KEY=...`) or anything else.
63

65	
66	- A training job that's too big or slow for your laptop / FarmShare.
67	- An evaluation that needs a particular GPU type and you don't want to wait in a queue.
68	- A many-config sweep where you want to spin up several containers in parallel.
69	- A one-off data preprocessing job that needs more RAM or disk than you have locally.

116	### About `--detach`
117	
118	For anything longer than about 10 minutes, run it with `--detach`:
119	
120	```

122	```

126	## The two-file pattern (for real training scripts)
127	
128	The template above puts your code inline inside a Modal function. That's fine for small one-off jobs, but for actual training scripts there's a cleaner pattern: **keep your training code as a regular Python file, and write a small Modal wrapper that calls it.**
129	
130	Two files instead of one:
131	
132	- **`train_my_thing.py`** — pure Python. argparse, training loop, save outputs. No Modal imports anywhere. You can run it locally on your laptop or on FarmShare with a small dataset for debugging.
133	- **`modal_train_my_thing.py`** — ~25-line Modal wrapper. Defines image, GPU, volume, then `subprocess.run`s the training script inside the container.
134	
135	**A working demo lives in this folder**: see `demo_train.py` (the "training" script) and `modal_demo.py` (the Modal wrapper that runs it).
136	
137	To try it:
138	```
139	modal run modal_demo.py
140	```
141	
142	That runs `demo_train.py` on a Modal container for ~2 seconds, writes a result file to the shared Volume at `/data/demo_output/`, and exits.
143	
144	To make your own version:
145	
146	1. Copy `modal_demo.py` to `modal_<your_thing>.py`.
147	2. Write your real training script as a sibling Python file (e.g., `train_eeg_tokenizer.py`).
148	3. In your new Modal wrapper file:
149	   - Change `add_local_python_source("demo_train")` to your script's module name.
150	   - Update the `subprocess.run([...])` command to pass the right args.
151	   - Set `gpu=`, `timeout=`, and any secrets you need.
152	4. `modal run modal_<your_thing>.py`.
153	
154	**Why this is nice:**
155	- Your training code stays portable — it runs anywhere Python runs.
156	- The Modal wrapper is small and rarely needs changes; you edit your training code freely.
157	- You can test your training locally with a tiny dataset before paying for Modal compute.
158	
159	IMPORTANT: Always set `timeout=` so a job can't run indefinitely.
160	
161	## Things to know before you click "run"
162	
163	- **Use the cheapest GPU that fits.** OOM errors are obvious; spending too much isn't. Start at T4 or A10, upgrade only when needed.
164	- **Run `modal_app.py::hello` first** in your workspace to confirm setup works.
165	- **Detached jobs keep running after you close your terminal.** Check the dashboard if you think you forgot one.
166	- **Anything outside `/data` disappears when the container ends.** Don't save to `/root/` or `/tmp/` expecting it to stick around.
167	- **Call `data_volume.commit()`** at the end of any function that writes to `/data`, or your writes won't persist to the Volume.
168

"""Download THINGS-MEG **preprocessed** data (figshare file 39472855, ~96 GB)
into /data/things-meg/preprocessed/ on the shared `neural-fm-data` Volume.

We swapped off the OpenNeuro S3 raw sync because the preprocessed
epochs are what we actually train on; the raw recordings are 4x bigger
and we'd just re-derive these epochs from them anyway.

Bytes transfer figshare -> Modal directly. Your laptop's only job is
to type `modal run --detach ...`.

Resumability: wget's `-c` flag resumes a partial transfer if the
container dies mid-download, so a re-run picks up where it stopped
(important since 96 GB over a flaky link is not zero-risk).

Run:
    modal run --detach modal_download_things_meg.py::download
"""

import os
import pathlib
import subprocess

from modal_app import app, data_volume, image

# apt: wget for the download, unzip in case the archive is a .zip.
# add_local_python_source: ships modal_app.py into the container so the
# `from modal_app import ...` line above resolves at remote-runtime.
download_image = (
    image
    .apt_install("wget", "unzip")
    .add_local_python_source("modal_app")
)

FIGSHARE_URL = "https://figshare.com/ndownloader/files/39472855"
DEST_DIR = "/data/things-meg/preprocessed"
STAGING_DIR = "/data/things-meg/_archive_staging"


@app.function(
    image=download_image,
    volumes={"/data": data_volume},
    # Network-bound. CPU-only; a GPU would idle while burning credits.
    cpu=4.0,
    memory=8 * 1024,
    # 6 hours. Sync is resumable, so a timeout is recoverable.
    timeout=60 * 60 * 6,
)
def download():
    """wget the figshare archive, extract into DEST_DIR, clean up."""
    os.makedirs(DEST_DIR, exist_ok=True)
    os.makedirs(STAGING_DIR, exist_ok=True)

    # --content-disposition: honor the filename figshare sends in the
    #     response header (otherwise wget would save it as "39472855").
    # -c: resume partial transfer on re-run.
    # --no-verbose: one line per file, no per-chunk progress spam.
    subprocess.run(
        [
            "wget",
            "--content-disposition",
            "-c",
            "--no-verbose",
            "-P", STAGING_DIR,
            FIGSHARE_URL,
        ],
        check=True,
    )

    # Identify what landed. Figshare gives us one file; assert that.
    archives = sorted(pathlib.Path(STAGING_DIR).iterdir())
    if len(archives) != 1:
        raise RuntimeError(
            f"Expected exactly one file in {STAGING_DIR}, got: {archives}"
        )
    archive = archives[0]
    print(f"Downloaded: {archive.name} ({archive.stat().st_size / 1e9:.1f} GB)")

    # Extract based on suffix; if it's not a recognized archive format,
    # just move the file into DEST_DIR as-is.
    name = archive.name.lower()
    if name.endswith(".zip"):
        subprocess.run(["unzip", "-q", str(archive), "-d", DEST_DIR], check=True)
        archive.unlink()
    elif name.endswith((".tar", ".tar.gz", ".tgz", ".tar.bz2", ".tar.xz")):
        subprocess.run(["tar", "-xf", str(archive), "-C", DEST_DIR], check=True)
        archive.unlink()
    else:
        print(f"Unrecognized archive format; leaving {archive.name} in place.")
        target = pathlib.Path(DEST_DIR) / archive.name
        archive.rename(target)

    # Clean up staging dir if empty.
    staging = pathlib.Path(STAGING_DIR)
    if staging.exists() and not any(staging.iterdir()):
        staging.rmdir()

    # Sanity check: final size + top-level structure in the log.
    subprocess.run(["du", "-sh", DEST_DIR], check=True)
    subprocess.run(["ls", "-la", DEST_DIR], check=True)

    data_volume.commit()

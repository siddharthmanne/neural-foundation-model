Good thing you asked — your mental model is almost right, but there's one catch in how I wrote it (and how stock 4M works too).

Short answer: no, it would still validate once — on the final epoch

My in-loop trigger (in_loop_val.py:55) is:

at_cadence = (epoch + 1) % eval_freq == 0 or (epochs and epoch + 1 == epochs)

With epochs: 10, eval_freq: 20:

epoch (0-idx) (epoch+1) % 20 == 0? last epoch? validates?
0–8 no no no
9 no (10 % 20 = 10) yes yes ← fires here

So the periodic cadence never hits, but the epoch + 1 == epochs clause forces one validation at the end. Stock 4M's own single-task eval does the exact same thing (run_training_4m.py:648) — ... or epoch + 1 == args.epochs — so even the stock eval would run once on the last epoch (since your 4m_things_data.yaml has a val: section).

That final-epoch rule is intentional: you almost always want a validation on the finished model. But it means eval_freq alone can't give you zero validation.

To truly run with NO validation

Pick based on how clean you want it:

  • Disable the named suite: comment out in_loop_val_tasks in 4m_things_main.yaml. That kills my in-loop suite entirely (no final-epoch run either).
  • Also disable stock's single-task eval: comment out the val: block in 4m_things_data.yaml so data_loaders_val is None. Otherwise stock still does one cheap eval at the very end.

Do both → zero validation regardless of eval_freq.

Optional: make eval_freq the sole control

If you'd rather have eval_freq fully govern it (so eval_freq > epochs = never, no special-casing the last epoch), I can drop the epoch + 1 == epochs clause from my wrapper. Trade-off: you'd then need eval_freq to land on the last epoch to ever get a final-model validation. Want that change?

Why it ends up there

The val numbers ride into the log through the train-stats dict:

  1. My wrapper merges the suite results into train_one_epoch's return — stats = {**stats, **suite_stats} (in_loop_val.py:59).
  2. The stock loop builds the log record from that return — log_stats = {**train_stats, 'epoch': epoch, ...} (run_training_4m.py:640).
  3. The stock loop appends it to disk — open(output_dir/"log.txt", "a").write(json.dumps(log_stats)+"\n") (run_training_4m.py:667-669).

A few specifics worth knowing:

  • One line per epoch, appended (mode="a"), only on epochs where val actually fired (the eval_freq cadence + final epoch). Epochs without val just omit the [in-loop val: …] keys.
  • Main process only — the write is guarded by is_main_process() (line 667), so multi-GPU runs log once, not per-rank.
  • log.txt is richer than stdout. The console print shows just each task's total loss; log.txt keeps every per-modality sub-loss too.
  • Location: {output_dir}/log.txt on the volume — for your real config that's /project/runs/4m_things_neural/log.txt. (If log_wandb: true, the same log_stats dict goes to wandb as well.)

So during a real training run you can tail that file (or pull it with modal volume get project runs/4m_things_neural/log.txt -) and see val-per-task + tokens-seen accumulating epoch by epoch.

    """Average per-target loss over up to n_batches val batches.

    Loss is reported ONLY on the task's ``out_domains`` (its prediction targets).
    The model may carry extra decoder heads — e.g. ``tok_rgb`` when only depth is
    this task's target — whose loss is 0 and would otherwise dilute the average
    (the bug: rgb2depth showing (depth + 0)/2). The task ``loss`` is the mean over
    ``out_domains`` alone, so a single-target task reports exactly that target.
    """
    model.eval()
    out_set = set(out_domains)
    totals: dict[str, float] = {}
    count = 0
    for i, x in enumerate(loader):
        if n_batches and i >= n_batches:
            break
        mod_dict = {
            mod: {k: v.to(device) for k, v in d.items()}
            for mod, d in x.items()
            if mod in all_domains
        }
        autocast = torch.autocast(device_type=device.type, dtype=dtype, enabled=dtype != torch.float32)
        with autocast:
            _, mod_loss = model(mod_dict, num_encoder_tokens=n_in,
                                num_decoder_tokens=n_out, loss_type=loss_type)
        target_losses = {mod: float(l.mean()) for mod, l in mod_loss.items() if mod in out_set}
        for mod, value in target_losses.items():
            totals[f"{mod}_loss"] = totals.get(f"{mod}_loss", 0.0) + value
        if target_losses:
            totals["loss"] = totals.get("loss", 0.0) + sum(target_losses.values()) / len(target_losses)
        count += 1
    if count == 0:
        return {"loss": float("nan"), "n_batches": 0}
    stats = {k: v / count for k, v in totals.items()}
    stats["n_batches"] = count
    return stats

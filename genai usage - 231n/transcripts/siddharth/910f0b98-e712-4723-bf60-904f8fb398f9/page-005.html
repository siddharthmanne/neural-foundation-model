def get_pcc(rec: torch.Tensor, raw: torch.Tensor):
    # B C W
    B, C, W, D = rec.shape
    x = rearrange(rec, "B C W D->(B C W) 1 D")
    y = rearrange(raw, "B C W D -> (B C W) 1 D")
    c = (
        (x - x.mean(dim=-1, keepdim=True))
        @ ((y - y.mean(dim=-1, keepdim=True)).transpose(1, 2))
        * (1.0 / (D - 1))
    ).squeeze()
    sigma = (torch.std(x, dim=-1) * torch.std(y, dim=-1)).squeeze() + 1e-6
    return (c / sigma).mean()

The FFT loss is real and properly implemented (hamming windowed rFFT, L1 on amplitude + L1 on phase). I was wrong to say "reconstruction-only" — the actual objective is much richer. Let me lay out what this means.

The correct picture of BrainOmni's loss

5 terms, equally summed (except phase at 0.5×):

Term Domain What it rewards
time_loss time L2 between reconstructed and target waveforms
exp(-pcc) time Per-channel Pearson correlation (differentiable surrogate; lower is better)
commitment_loss latent Encoder outputs commit to nearest codebook entry (standard VQ-VAE)
amp_loss freq L1 between FFT magnitude spectra
phase_loss freq L1 between FFT phase spectra (half-weighted)

This is essentially the standard neural-audio-codec training recipe (SEANet-style, similar to EnCodec/SoundStream) — it's a well-engineered objective for waveform compression, not a naive MSE.

So my "reconstruction-only → need spectral loss" framing was wrong

The §5.3 category failure can't be attributed to a missing FFT term. It's already there. The recommendation "switch to a tokenizer with spectral loss" makes no sense — BrainOmni is that tokenizer.

What actually likely explains the category-info loss

If the objective is fine, the bottlenecks must be elsewhere. Working hypotheses, in rough likelihood order:

  1. Compression ratio is brutal. 271 ch × 281 samples = 76,151 input dims → 16 latent neurons × 8 temporal tokens × 4 RVQ × 9 bits ≈ 1,152 bits per trial. That's ~600× compression. Even a perfect lossy codec has to drop something at that ratio, and reconstruction loss budgets bits to whatever dominates variance — which on MEG is subject-level / low-frequency drift, not the fine spatial-temporal patterns that encode object category. (This is consistent with our finding that subject decodes at 62% but category at 3% — the bits got spent on subject-level signal.)

  2. The 16-latent-neuron spatial bottleneck. BrainOmni collapses 271 channels into 16 latents via cross-attention. THINGS-MEG decoding papers consistently report that channel-topographic patterns carry object-category info (Cichy et al., Hebart et al.). Collapsing 271 → 16 likely throws away the relevant spatial dimensions.

  3. Pretraining distribution mismatch. SEANet encoder/decoder were pretrained on 656h of resting-state EEG. We're finetuning under adapt mode (those convs frozen). Resting-state EEG has continuous oscillatory structure; event-related MEG has short-latency transients with very different spectral signatures. The pretrained convs may not transfer.

  4. FFT loss is global, but task-relevant info lives in narrow frequency bands. The L1 on FFT magnitudes treats every frequency bin equally. But MEG object-category info concentrates in α/β band (8-30 Hz) and the evoked response window (0.1-0.4s post-stimulus). Equal-weighted L1 over the full 1-100 Hz × 1.4s spectrum lets the loss be minimized by matching dominant low-frequency / DC components — exactly the subject-level signal.

Revised Stage 2 recommendation

The previous "switch to a tokenizer with FFT loss" path is dead. The new candidates:

  • Less aggressive compression: wider latent bottleneck (e.g., 64 or 128 latents instead of 16). Same architecture, fewer compressed dims.
  • Band-weighted FFT loss: weight α/β band higher than DC/delta. Cheap modification to the existing loss.
  • Auxiliary contrastive / supervised loss: add a category-discriminative term during finetune (e.g., contrastive InfoNCE pairing same-image trials, or direct supervised cross-entropy on the 27-way labels). This is the most direct path — gives the tokenizer a gradient signal that's specifically about the thing we're measuring.
  • Cho2026 / EphysTokenizer: still on the table, but their advantage isn't "spectral loss" — it's a different architectural family (smaller compression, no SEANet pretraining baggage). Less compelling than before.

The auxiliary loss path is the most directly addressed at the diagnosed failure. The FFT/spectral toolkit is already maxed out.

## CNN probe head — design

The flat MLP probe has no inductive bias for the structure that THINGS-MEG decoding actually exploits (Cichy / Hebart papers: channel-topographic patterns over time). A 2-layer MLP on flattened tokens may simply lack the right shape; per Dixen 2024 even on EEG, EEGNet/Conformer-style temporal convolutions massively outperform flat heads at this SNR.

The new `cnn` head exploits the token tensor's natural structure rather than flattening it:

- **For BrainOmni tokens** of shape `(B, 16 latents, 8 time, D=512 embed)`: a small **2-D CNN** with conv over `(latent × time)` and `D` as channels. Two conv blocks with batch-norm + ReLU + global average pooling → linear classifier. ~150k params.
- **For raw MEG** of shape `(B, 271 channels, 281 time)`: a small **1-D CNN** over time, channels as input dims. Two strided conv blocks → global average pooling → linear. ~50k params.
- **For random tokens**: same shape transform as `tokens` (random codes → `tokens_to_embedding` → CNN). Tests "what does the CNN do with uninformative tokens of the right shape?".
- **For μ-transform-style position-preserving tokens** `(B, 1, channels × time)`: reshape to `(B, channels, time)` then use the 1-D CNN — equivalent to applying the raw-MEG CNN to the dequantized signal.

Rationale: small enough to not overfit on ~4k training trials per fold, while giving the probe the same inductive bias (temporal smoothness + latent-position locality) that real MEG decoders use.

CNN design — locking in the architecture before coding

Three input shape families:

Source Natural shape (after batch) Dispatch target
BrainOmni tokens (RVQ-decoded) (16 latent, 8 time, D=512 embed) CNN2D — D as channels, conv over (latent × time)
μ-transform tokens (dequantized) (271 channels, 281 time) CNN1D — channels as input dim, conv over time
Raw signal (271 channels, 281 time) CNN1D — same as μ-transform
Random tokens matches real tokens shape same dispatch as tokens

Dispatch rule: 3D feats after batch ((B, C, T)) → CNN1D. 4D feats ((B, S1, S2, D)) → CNN2D.

CNN1D head (~33k–145k params depending on input)

Conv1d(C_in, 64, k=7, stride=2, padding=3) → BN → ReLU
Conv1d(64,    64, k=5, stride=2, padding=2) → BN → ReLU
AdaptiveAvgPool1d(1) → Flatten
Linear(64, n_classes)

Large initial kernel (7) + stride-2 captures local temporal pattern then downsamples. Two convs total; we're probing, not learning the whole task.

CNN2D head (~333k params for D=512)

Conv2d(D_embed, 64, k=(3,3), padding=1) → BN → ReLU
Conv2d(64,      64, k=(3,3), padding=1) → BN → ReLU
AdaptiveAvgPool2d((1,1)) → Flatten
Linear(64, n_classes)

3×3 spatial kernels over the (16 latent × 8 time) grid. Smaller field-of-view than CNN1D because the grid itself is small.

Why this is a fair "right inductive bias" probe

  • Temporal smoothness assumption (convs over time): consistent with what EEGNet/Conformer use on MEG/EEG
  • Latent locality on the 2D grid: weak but defensible (some latents in BrainOmni learn similar functions)
  • Small parameter count: ~150k vs MLP's 20M for raw — much less overfitting risk
  • No attention / recurrence: keeps the §5.3 spirit ("simplest probe that has the right inductive bias")

            `(D,)` for flat features; `(C, T)` for 1-D time-series;
            `(S1, S2, D)` for 2-D structured embeddings.
        n_classes: classification target size.
        config: EvalConfig.

    Heads:
        "linear" — §5.3 gate (what 4M's input embedding effectively does).
                   Flatten + Linear.
        "mlp"    — 2-layer ReLU+dropout. Tests "info is there but nonlinear."
        "cnn"    — Conv1d head (3-D inputs) or Conv2d head (4-D inputs).
                   Tests "info needs the right inductive bias to surface."
    """
    feat_dim = int(torch.tensor(feat_shape).prod().item())
    clf = config.probe_classifier
    if clf == "linear":
        return torch.nn.Sequential(
            torch.nn.Flatten(),
            torch.nn.Linear(feat_dim, n_classes),
        )
    if clf == "mlp":
        return torch.nn.Sequential(
            torch.nn.Flatten(),
            torch.nn.Linear(feat_dim, config.probe_mlp_hidden),
            torch.nn.ReLU(),
            torch.nn.Dropout(config.probe_mlp_dropout),

        )

    raise ValueError(
        f"probe_classifier must be 'linear', 'mlp', or 'cnn'; got {clf!r}"

def _build_cnn_head(
    feat_shape: tuple[int, ...], n_classes: int, hidden: int
) -> torch.nn.Module:
    """Shape-dispatched CNN head.

      - (C, T)      → CNN1D: channels-as-input, conv over time.
      - (S1, S2, D) → CNN2D: D-as-input, conv over (S1, S2).

    Other shapes (e.g. flat or 4-D non-token) are rejected — they don't have
    a natural conv interpretation and the user should switch to linear/mlp.
    """
    if len(feat_shape) == 2:
        in_channels, _t = feat_shape
        return _CNN1DHead(in_channels=in_channels, n_classes=n_classes, hidden=hidden)
    if len(feat_shape) == 3:
        _s1, _s2, d_embed = feat_shape
        return _CNN2DHead(in_channels=d_embed, n_classes=n_classes, hidden=hidden)
    raise ValueError(
        f"CNN head expects features with shape (C, T) or (S1, S2, D); "
        f"got {feat_shape!r}"
    )


class _CNN1DHead(torch.nn.Module):
    """Conv1d → BN → ReLU ×2 → AdaptiveAvgPool1d → Linear.

    Designed for (B, C, T) — raw MEG (C=271, T=281) or μ-transform
    dequantized tokens (same shape).
    """

    def __init__(self, in_channels: int, n_classes: int, hidden: int = 64):
        super().__init__()
        self.net = torch.nn.Sequential(
            torch.nn.Conv1d(in_channels, hidden, kernel_size=7, stride=2, padding=3),
            torch.nn.BatchNorm1d(hidden),
            torch.nn.ReLU(),
            torch.nn.Conv1d(hidden, hidden, kernel_size=5, stride=2, padding=2),
            torch.nn.BatchNorm1d(hidden),
            torch.nn.ReLU(),
            torch.nn.AdaptiveAvgPool1d(1),
            torch.nn.Flatten(),
            torch.nn.Linear(hidden, n_classes),
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:  # (B, C, T) → (B, n_classes)
        return self.net(x)


class _CNN2DHead(torch.nn.Module):
    """Conv2d → BN → ReLU ×2 → AdaptiveAvgPool2d → Linear.

    Designed for (B, S1, S2, D) — BrainOmni tokens decoded through
    `tokens_to_embedding` and kept structured: S1=16 latent, S2=8 time,
    D=embedding dim. The forward pass permutes D into the channel slot.
    """

    def __init__(self, in_channels: int, n_classes: int, hidden: int = 64):
        super().__init__()
        self.net = torch.nn.Sequential(
            torch.nn.Conv2d(in_channels, hidden, kernel_size=3, padding=1),
            torch.nn.BatchNorm2d(hidden),
            torch.nn.ReLU(),
            torch.nn.Conv2d(hidden, hidden, kernel_size=3, padding=1),
            torch.nn.BatchNorm2d(hidden),
            torch.nn.ReLU(),
            torch.nn.AdaptiveAvgPool2d((1, 1)),
            torch.nn.Flatten(),
            torch.nn.Linear(hidden, n_classes),
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:  # (B, S1, S2, D)
        x = x.permute(0, 3, 1, 2).contiguous()
        return self.net(x)

You're 100% right and I was about to set this up wrong. Let me think this through and propose a clean design.

Why train-eval consistency matters here

If we train BrainOmni on averaged MEG, the model's encoder learns the statistics of averaged signal (lower noise floor, different spectral profile, smoother spatial structure). Feeding single-trial MEG to that encoder at eval time is an out-of-distribution input — the model would see signal it wasn't trained for, and any failure or success couldn't be attributed to the tokenizer cleanly.

So train averaged → eval averaged is non-negotiable. Anything else conflates the experiment.

But this creates a real design tension

Cross-subject + cross-rep averaging gives the cleanest noise reduction (one sample per unique image), but it also eliminates subject as a decodable variable — by definition, an averaged trial has no subject identity. Three implications:

  1. Subject-ID is N/A for 3b_avg evaluation (the task doesn't exist on this data). We lose the pipeline-sanity diagnostic that was so valuable in Experiment 1.
  2. Eval split size drops 4×: ~10k test trials → ~2.5k averaged samples → ~1.2k after dropping unmapped concepts. Fewer trials per fold means wider std bars; the 27-way category at chance signal we're hunting for has very small absolute effect (raw at ~5%) so this matters.
  3. Train data drops 4×: 88k → ~18k. Workable for adapt (959k params → ~1.4k supervision values per param, below current ~7k but in the comfortable zone) but rules out full (5M params would be in the overfitting zone).

Three averaging strategies — pick one

Option Averaging Train samples Subject-decodable? SNR gain
A — cross-subject + cross-rep All trials of an image averaged into one ~18k ❌ no 2× for exp (1 rep × 4 subj), ~7× for test (12 reps × 4 subj)
B — within-subject + cross-rep All reps of (image, subject) averaged ~89k ✅ yes (subject preserved) ~1× for exp (already 1 rep per subject), ~3.5× for test (12 reps → 1 per subject)
C — only test images averaged within subject exp images untouched, test images rep-averaged ~89.8k ✅ yes Only affects 200 test images × 4 subjects = 800 samples

My recommendation: B

Option B is the cleanest experiment because: - It directly tests the hypothesis "did per-trial noise hurt category preservation?" without confounding by data quantity (train size barely changes) - All 3 tasks remain valid — subject, animacy, category27 all decodable - Statistical power preserved — eval split stays close to ~10k → ~5k usable - Apples-to-apples with 3b_nonavg on the same metric grid

Option A is the most aggressive but creates real interpretive problems. Option C barely changes anything.

The downside of B: the SNR boost is small for exp images (which are 90% of data). Test images get a bigger gain but they're rare. So B might show only a small lift on category if any.

If B shows nothing, A becomes worth running as the "if averaging is going to help, this is the strongest version" experiment, accepting that subject decoding drops out.

def average_trials_by_image(
    X: torch.Tensor | np.ndarray,
    image_ids: np.ndarray,
) -> tuple[torch.Tensor, np.ndarray]:
    """Group trials by `image_id` and average within each group.

    Produces one signal per unique image — the canonical "cross-subject +
    cross-rep" averaging used by Experiment 2 (see
    ``notes/meg_tokenization.md``). The averaging axis collapses both
    subject and within-subject repetitions, so the output has no subject
    label per row.

    Args:
        X: ``(N, C, T)`` torch or numpy array of single-trial signals.
        image_ids: ``(N,)`` int array, trial-aligned image codes.

    Returns:
        X_avg:     ``(M, C, T)`` torch.float32 tensor (M = unique image
                   count).
        image_ids_unique: ``(M,)`` int64 numpy array, sorted ascending.

    The mean is computed in float64 to keep numerical accuracy when a
    single image has 40+ test-set reps × subjects; casts to float32 at
    return.
    """
    if isinstance(X, torch.Tensor):
        X_np = X.cpu().to(torch.float64).numpy()
    else:
        X_np = np.asarray(X, dtype=np.float64)

    image_ids = np.asarray(image_ids, dtype=np.int64)
    if X_np.shape[0] != image_ids.shape[0]:
        raise ValueError(
            f"X.shape[0]={X_np.shape[0]} must match image_ids length "
            f"{image_ids.shape[0]}"
        )

    unique_ids, inverse = np.unique(image_ids, return_inverse=True)
    n_groups = len(unique_ids)
    C, T = X_np.shape[1], X_np.shape[2]

    sums = np.zeros((n_groups, C, T), dtype=np.float64)
    counts = np.zeros(n_groups, dtype=np.int64)
    np.add.at(sums, inverse, X_np)
    np.add.at(counts, inverse, 1)

    X_avg_np = sums / counts.reshape(n_groups, 1, 1)
    return torch.from_numpy(X_avg_np.astype(np.float32)), unique_ids.astype(np.int64)

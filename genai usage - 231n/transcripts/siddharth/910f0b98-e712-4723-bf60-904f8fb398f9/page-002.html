Great question — the right answer is don't subsample to balance the training set. There's a simpler, statistically more powerful fix.

What you're actually asking

The probe's question is: "do the tokens carry information that distinguishes object categories?"

Top-1 accuracy with imbalanced classes can't answer that — it conflates "decoded a class" with "predicted the majority class." Two separate things you can do to fix it:

Option A: Fix the metric (recommended, simpler)

Train normally on all data, but report balanced accuracy instead of (or alongside) top-1.

Balanced accuracy = mean(per_class_recall). It's invariant to class imbalance: - Majority-class predictor: gets food at 100%, every other class at 0% → balanced acc = 1/27 = 3.7% - A classifier that actually distinguishes classes: balanced acc moves above 3.7%

That's the diagnostic you want. The chance line sits at the value you expected.

Option B: Fix the training (subsample to balance, what you proposed)

Subsample so every class has the same N trials in training, then train, then evaluate.

This works, but: - Throws away data — you have to subsample to your smallest class. With class counts ranging 14 → 123 concepts, you'd cap at ~14 × trial_factor = ~80 trials/class × 27 ≈ 2,200 trials usable instead of 5,032. ~55% data loss. - High variance — your test result depends on which random subset you happened to pick. - You still need to pick a metric. Even with balanced training, the metric question doesn't go away.

Option C: Fix the training without subsampling (class-weighted loss)

Weight each class by 1/frequency in the cross-entropy loss. Same effect as B in expectation, no data loss.

Why A alone is usually enough

The classifier's behavior is determined by both features and training distribution. But the diagnostic question — is there decodable signal — is fully answered by balanced accuracy on a normally-trained probe:

  • Features have no signal → classifier predicts majority → balanced acc = 1/27 ≈ 3.7%
  • Features have signal → classifier learns to use it → balanced acc > 3.7%
  • Features have weak signal in some classes only → balanced acc > 3.7% by a small amount (still detectable)

You don't need to artificially help the classifier with balanced training to see whether the signal exists. If the signal is linearly decodable at all, balanced accuracy will lift above chance.

When you'd want B or C anyway

If balanced accuracy with normal training shows ~3.7% (i.e. no signal at all), that's the final answer — there's no signal. No amount of training tricks rescues that.

If balanced accuracy shows a small bump (say 5-7%), you might want to add class-weighted training (Option C) as a second variant to see if the small bump grows when minority classes get more gradient signal. But this is a secondary question.

Subsampling (Option B) is mainly useful for publication-grade claims where reviewers might object "you only got 12% because of imbalance." For diagnostic use, B is strictly worse than A.

My recommendation

Do A first — it's a ~15-line change to probe.py and immediately tells you whether BrainOmni's tokens carry any decodable signal. If it shows signal, you have your answer. If it doesn't, the conclusion is robust and you don't need to chase fixes that won't help.

The change:

# in _topk_accuracy or a new _balanced_accuracy helper
def _balanced_accuracy(logits, labels, n_classes):
    preds = logits.argmax(dim=1)
    per_class_recall = []
    for c in range(n_classes):
        mask = labels == c
        if mask.any():
            per_class_recall.append((preds[mask] == c).float().mean().item())
    return float(sum(per_class_recall) / len(per_class_recall))

Add it to the probe output as bal_acc_{set}_mean / _std. Same forward pass, just one more aggregation.

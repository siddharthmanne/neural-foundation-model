  • In neural_masking.py:56-70, zero_absent_budgets sets the MEG/EEG Dirichlet budget to 0 for placeholder samples but does not redistribute those tokens to other modalities.

Volume is project, so your run lands at /project/runs/4m_things_neural/. Here's exactly where the token info goes, in order of how soon you see it.

1. Live console (stdout / Modal logs) — every epoch

The in-loop-val hook prints it next to the val losses (in_loop_val.py:64-71). With your config (eval_freq: 1, in_loop_val_tasks set), that's every epoch:

[in-loop val] epoch 0: [in-loop val: rgb2depth] loss=8.93  ...
[in-loop val] epoch 0: tokens seen (training): input=22,016 target=18,300 total=40,316

This streams live while modal run ... is going. Caveat: this line is emitted by the in-loop-val path, so it only shows if in_loop_val_tasks is set. Turn that off and the console goes quiet on tokens (it'd only be in log.txt).

2. log.txt — every epoch, persistent

The stock trainer writes one JSON line per epoch to {output_dir}/log.txt (run_training_4m.py:667-669). Our wrapper merges the accurate counts into that line, so at /project/runs/4m_things_neural/log.txt you get:

{"[Epoch] loss": ..., "epoch": 0,
 "[tokens] input_seen": 22016, "[tokens] target_seen": 18300, "[tokens] total_seen": 40316,
 "input_tokens_seen_b": 0.000131, "target_tokens_seen_b": ...}

Note both live side by side: [tokens] *_seen (our accurate raw counts) and stock's *_tokens_seen_b (the closed-form, in billions — the one that overcounts placeholders). You can compare them directly.

3. Inside each checkpoint .pth — every 10 epochs

install_token_accounting stashes the running total onto args.tokens_seen, and stock save_model serializes args into the file (checkpoint.py:100-105). So /project/runs/4m_things_neural/checkpoint-9.pth, -19.pth, …, checkpoint-final.pth each carry the tokens-seen for that model state. Read it with read_tokens_seen(torch.load(path, weights_only=False)).

4. Standalone validation — reads it back from the checkpoint

modal run 4m_training/modal/modal_train.py --validate (or validate_4m.py) prints tokens seen (training): ... for whatever checkpoint it loads (validate_4m.py:267).

5. wandb — only if you enable it

Your config has log_wandb: false. Flip it to true and the [tokens] *_seen keys ride along in log_writer.update(log_stats) as epoch-level metrics. (The per-step wandb token log at run_training_4m.py:776-783 is stock's overcounting formula only — our accurate number is per-epoch / per-checkpoint, not per-step.)

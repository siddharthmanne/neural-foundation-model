Short answer: yes, it's structurally guaranteed, because the trial is sampled exactly once in the data loader and that one tensor becomes both the embedded input and the loss target. There is no second sampling between the forward pass and backprop. Let me trace it concretely so you can see why.

Where the trial is picked: once per sample, in the data-loading transform (CPU, in the DataLoader worker). The result is a single token array T for that modality in that batch element. The model never re-samples — T is frozen for the entire forward/backward step.

How T becomes the target. For a parallel (img / our neural_grid) modality, the decoder embedding's forward_embed does this (decoder_embeddings.py:242-254):

ids = d['tensor']          # <-- the sampled trial T
ids = ids.reshape(B, -1)
x   = self.token_emb(ids)
d['x']   = x
d['ids'] = ids             # <-- target ids ARE the same T

Then in cat_decoder_tensors (fm.py:322-323):

decoder_tokens_all.append(torch.zeros_like(d['x']) + self.mask_token)  # input = MASK, not T
target_ids_all.append(d['ids'])                                        # label  = T

and the loss (fm.py:597) is cross_entropy(logits, target_ids) over exactly those d['ids']. So:

  • The decoder input at a target position is the mask token (so there's no leakage — the model can't just copy the answer).
  • The label at that position is T — the very trial that was sampled.
  • They are the same array object, set in the same forward_embed call. Nothing can desync them.

The only desync risk is the one we engineered around: the 4 MEG RVQ layers are 4 separate modalities. If each ran its own trial-sampling transform, layer 0 might be trial 7 and layer 2 trial 3 — incoherent across heads. That's exactly why the plan picks the trial once in the shared _rename_modalities seam and slices all 4 layers from that single pick. Within each head, the "input-embedding vs. loss-target" coherence above is automatic; across heads, the single shared pick guarantees they describe the same MEG token.
